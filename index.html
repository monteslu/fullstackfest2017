Web RTC

Peter wants to have a chat with his good friend Varys, but he has trust issues and he doesn't want to use Facebook Messenger or Slack...

Web RTC

He decides to use Web RTC PeerConnection DataChannel, so his web browser of choice makes a request to a public STUN server and creates an offer packet.
He needs to get that offer message over to Varys so that Varys's browser can figure out how to make a direct connection to him.

Web RTC

While Peter knows a signaling server that uses websockets can be great for this sort of thing, nothing is faster than a Raven.

Web RTC

Varys gladly types the offer into his browser byte by byte, and also using a STUN server he creates an answer message, which he copies and pastes into an email because his handwriting isn't as nice as Peter's.

Web RTC

Peter then uses Vary's answer to create a direct peer-to-peer connection. While he could attach voice and video to this secure connection, he's going to use it for chat messages.

PeerConnection DataChannel !

Peter and Varys can both easily listen for messages.
